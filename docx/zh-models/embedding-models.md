# 嵌入模型

## 概述

Kelivo 支持两种类型的 AI 模型：**聊天模型**和**嵌入模型**。本指南将解释什么是嵌入模型以及如何配置它们。

## 什么是嵌入模型？

嵌入模型是专门将文本转换为数值向量（嵌入）的 AI 模型。与生成对话响应的聊天模型不同，嵌入模型主要用于：

- **语义搜索**：基于含义查找相似文本
- **文本分类**：对内容进行分类
- **推荐系统**：推荐相关项目
- **聚类分析**：将相似文档分组

### 示例

常见的嵌入模型包括：
- Google：`text-embedding-004`、`text-embedding-005`
- OpenAI：`text-embedding-3-small`、`text-embedding-3-large`、`text-embedding-ada-002`
- 名称中包含 "embed" 或 "embedding" 的模型

## 与聊天模型的区别

| 特性 | 聊天模型 | 嵌入模型 |
|------|---------|---------|
| **用途** | 生成对话响应 | 将文本转换为向量 |
| **输入/输出** | 文本/图像 → 文本/图像 | 文本 → 数值向量 |
| **工具调用** | ✅ 支持 | ❌ 不支持 |
| **多模态** | ✅ 通常支持 | ❌ 仅文本 |
| **使用场景** | 对话、问答、生成 | 搜索、分类、相似度 |

## 配置嵌入模型

### 自动检测

对于 **Google** 提供商，Kelivo 会根据 API 功能自动检测嵌入模型：
- 具有 `embedContent` 方法 → 嵌入模型
- 具有 `generateContent` 方法 → 聊天模型

### 手动配置

1. 导航至 **设置** → **提供商** → 选择您的提供商
2. 点击 **添加模型** 或编辑现有模型
3. 配置以下内容：
   - **模型 ID**：API 标识符（例如：`text-embedding-004`）
   - **显示名称**：人类可读的名称（例如：`Gemini 嵌入`）
   - **类型**：选择 **"嵌入"** 而不是 "聊天"
4. 保存配置

### UI 简化

当您将模型类型设置为 "嵌入" 时，配置界面会简化：

**可见选项：**
- ✅ 模型 ID
- ✅ 显示名称
- ✅ 类型选择（聊天/嵌入）
- ✅ 自定义请求头（高级）
- ✅ 自定义请求体参数（高级）

**隐藏选项：**
- ❌ 输入模式（文本/图像）
- ❌ 输出模式（文本/图像）
- ❌ 能力（工具调用/推理）

这些选项被隐藏是因为嵌入模型不支持对话功能、多模态输入/输出或工具调用。

## 模型分组

嵌入模型会在模型选择界面中自动分组到 "嵌入" 部分。这使得在浏览可用模型时，可以轻松区分聊天模型和嵌入模型。

## 何时使用嵌入类型

✅ **在以下情况下将模型设置为 "嵌入" 类型：**
- 该模型是为文本嵌入（向量生成）设计的
- 模型名称包含 "embed" 或 "embedding"
- 您想在 UI 中将其与聊天模型区分开
- 该模型不支持对话交互

❌ **在以下情况下不要设置为 "嵌入" 类型：**
- 它是聊天/对话模型（GPT、Gemini、Claude 等）
- 它支持工具调用或推理
- 它生成对提示的文本响应

## 配置示例

以下是嵌入模型在配置中的存储方式：

```json
{
  "apiModelId": "text-embedding-004",
  "name": "Gemini 文本嵌入",
  "type": "embedding",
  "input": ["text"],
  "output": ["text"],
  "abilities": []
}
```

## 提示

- 嵌入模型通常**不会直接用于**聊天对话
- 它们是用于特定技术用例的专用工具
- 如果不确定模型是否用于嵌入，请查看提供商的文档
- 大多数用户主要使用**聊天模型**进行日常交互
