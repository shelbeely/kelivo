# A2UI Quick Start Guide

## For Users

### What is A2UI?

A2UI (Agent-to-User Interface) is a new feature in Kelivo that allows AI agents to create interactive user interfaces beyond simple text responses. Instead of just reading text, you can now interact with forms, buttons, sliders, and other UI elements generated by the AI.

### How to Access the Demo

1. Open Kelivo
2. Tap the **Settings** icon (bottom navigation bar)
3. Look for **A2UI Demo** in the General section
4. Tap to open the demo page

### Demo Features

The demo page showcases four different types of interfaces:

1. **Simple Card** - Basic text display in a card
2. **Form Example** - Interactive booking form with input fields
3. **Settings Panel** - Toggles, switches, and sliders
4. **Layout Example** - Multi-column feature comparison

You can switch between examples using the chips at the top of the demo page. Each example shows:
- The rendered UI (how it looks)
- The JSON specification (how it's defined)

## For AI Agents

### Generating A2UI Interfaces

When responding to user requests, you can generate A2UI specifications in JSON format. Kelivo will automatically render these into interactive interfaces.

### Basic Structure

```json
{
  "type": "widget_type",
  "property1": "value1",
  "children": [...]
}
```

### Available Widgets

#### Card
Container with elevation and padding:
```json
{
  "type": "card",
  "padding": 16,
  "elevation": 2.0,
  "children": [...]
}
```

#### Text
Display styled text:
```json
{
  "type": "text",
  "text": "Your message here",
  "fontSize": 18,
  "bold": true,
  "color": "#FF0000"
}
```

#### Button
Interactive button:
```json
{
  "type": "button",
  "id": "unique_id",
  "label": "Click Me"
}
```

#### Text Field
Input field:
```json
{
  "type": "textfield",
  "id": "input_id",
  "label": "Name",
  "hint": "Enter your name"
}
```

#### Container
Layout container:
```json
{
  "type": "container",
  "padding": {"left": 8, "top": 8, "right": 8, "bottom": 8},
  "backgroundColor": "#F5F5F5",
  "borderRadius": 8,
  "children": [...]
}
```

#### Row / Column
Flex layouts:
```json
{
  "type": "row",
  "mainAxisAlignment": "spaceBetween",
  "children": [...]
}
```

```json
{
  "type": "column",
  "crossAxisAlignment": "start",
  "children": [...]
}
```

#### Divider
Visual separator:
```json
{
  "type": "divider",
  "thickness": 1.0,
  "color": "#E0E0E0"
}
```

#### Checkbox
Boolean selection:
```json
{
  "type": "checkbox",
  "id": "agree",
  "label": "I agree to terms",
  "value": false
}
```

#### Switch
Toggle switch:
```json
{
  "type": "switch",
  "id": "notifications",
  "label": "Enable notifications",
  "value": true
}
```

#### Slider
Numeric value selector:
```json
{
  "type": "slider",
  "id": "volume",
  "value": 0.5,
  "min": 0.0,
  "max": 1.0
}
```

### Complete Example

Here's a complete example of a restaurant booking interface:

```json
{
  "type": "card",
  "padding": 16,
  "children": [
    {
      "type": "text",
      "text": "Restaurant Booking",
      "fontSize": 24,
      "bold": true,
      "margin": {"bottom": 16}
    },
    {
      "type": "textfield",
      "id": "name",
      "label": "Name",
      "hint": "Your name"
    },
    {
      "type": "textfield",
      "id": "email",
      "label": "Email",
      "hint": "your@email.com"
    },
    {
      "type": "divider",
      "margin": {"top": 16, "bottom": 16}
    },
    {
      "type": "text",
      "text": "Number of Guests",
      "fontSize": 14,
      "bold": true
    },
    {
      "type": "slider",
      "id": "guests",
      "value": 2,
      "min": 1,
      "max": 10
    },
    {
      "type": "switch",
      "id": "outdoor",
      "label": "Outdoor seating",
      "value": false
    },
    {
      "type": "divider",
      "margin": {"top": 16, "bottom": 16}
    },
    {
      "type": "row",
      "mainAxisAlignment": "end",
      "children": [
        {
          "type": "button",
          "id": "book",
          "label": "Book Table"
        }
      ]
    }
  ]
}
```

### Best Practices

1. **Keep it simple**: Start with basic widgets and add complexity as needed
2. **Use proper IDs**: Give unique IDs to interactive elements for tracking
3. **Provide defaults**: Always include default values for inputs
4. **Add spacing**: Use margins and padding for better visual hierarchy
5. **Group related items**: Use cards and containers to group related UI elements
6. **Be accessible**: Always provide labels for form fields

### Property Reference

#### Common Properties

All widgets support:
- `margin`: Number or object `{left, top, right, bottom}`

#### Alignment Values

For `mainAxisAlignment`:
- `"start"` - Align to start
- `"center"` - Center alignment
- `"end"` - Align to end
- `"spaceBetween"` - Space between items
- `"spaceAround"` - Space around items
- `"spaceEvenly"` - Even spacing

For `crossAxisAlignment`:
- `"start"` - Align to start
- `"center"` - Center alignment
- `"end"` - Align to end
- `"stretch"` - Stretch to fill

#### Color Format

Colors can be specified as:
- Hex strings: `"#FF0000"` or `"#AARRGGBB"`
- Integers: `0xFFFF0000`

## Security

A2UI is designed to be secure:

✅ **Declarative only** - No executable code, just data
✅ **Pre-approved widgets** - Limited to safe, trusted components
✅ **Safe parsing** - Errors are caught and displayed safely
✅ **No dynamic code** - Widgets are statically defined

## Troubleshooting

### UI Not Rendering?

- Check JSON syntax (use a JSON validator)
- Ensure widget type is supported
- Check required properties are provided
- Look for error messages in the rendered UI

### Widget Looks Wrong?

- Verify property names and types
- Check color format (hex or integer)
- Ensure numeric values are numbers, not strings
- Review margin/padding structure
